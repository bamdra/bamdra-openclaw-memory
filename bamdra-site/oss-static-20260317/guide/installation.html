Installation ​

Supported install paths ​

The suite supports two public install paths:

  1. npm / OpenClaw CLI install
  2. manual install from the release zip

If your OpenClaw environment supports npm plugin install, prefer that. If you want a manual or offline path, use the release zip.

All three plugins can run independently. When bamdra-openclaw-memory is installed through npm, it will automatically provision bamdra-user-bind if that dependency has not yet been materialized into the local OpenClaw extensions directory. bamdra-memory-vector remains independent and optional.

Requirements ​

  • OpenClaw
  • Node.js 22 or newer
  • pnpm 10 or newer
  • Writable local path for SQLite

npm install ​

bash
openclaw plugins install @bamdra/bamdra-openclaw-memory

That install path should also auto-create the local memory directory and auto-provision bamdra-user-bind into ~/.openclaw/extensions/ when needed.

Then bind both plugins.slots.memory and plugins.slots.contextEngine to bamdra-openclaw-memory.

Add bamdra-memory-vector only when you want semantic retrieval enhancement:

bash
openclaw plugins install @bamdra/bamdra-memory-vector

What to download ​

  • Bamdra OpenClaw Memory release package v0.3.1
  • Local SQLite merge config
  • Full suite merge config
  • GitHub home
  • npm package
  • bamdra-user-bind repository
  • bamdra-memory-vector repository

Install steps ​

1. Unzip the release package ​

The package contains:

  • bamdra-openclaw-memory/
  • bamdra-user-bind/
  • bamdra-memory-vector/
  • examples/configs/
  • INSTALL.md

2. Prepare OpenClaw directories ​

The main plugin will create the missing memory directory on first bootstrap. You only need to make sure ~/.openclaw/extensions/ exists if you are manually copying folders.

3. Copy the plugin folder ​

Copy this directory into ~/.openclaw/extensions/:

  • bamdra-openclaw-memory

Optional enhancement:

  • bamdra-memory-vector

bamdra-user-bind can also be copied manually if you want it as a standalone plugin, but it is no longer a required second manual step for the main plugin install flow.

4. Merge config carefully ​

Do not overwrite the entire plugins object if your OpenClaw install already has other plugins enabled.

For the safest runtime compatibility, make sure your merged config does all of the following:

  • enables plugins globally
  • allows bamdra-openclaw-memory
  • loads the unified plugin path
  • sets both plugins.slots.contextEngine and plugins.slots.memory to bamdra-openclaw-memory
  • adds memory-core to plugins.deny
  • adds the config entry for bamdra-openclaw-memory

Example shape:

json
{
  "plugins": {
    "enabled": true,
    "deny": [
      "memory-core"
    ],
    "allow": [
      "bamdra-openclaw-memory"
    ],
    "load": {
      "paths": [
        "~/.openclaw/extensions/bamdra-openclaw-memory"
      ]
    },
    "slots": {
      "contextEngine": "bamdra-openclaw-memory",
      "memory": "bamdra-openclaw-memory"
    },
    "entries": {
      "bamdra-openclaw-memory": {
        "enabled": true,
        "config": {
          "enabled": true,
          "store": {
            "provider": "sqlite",
            "path": "~/.openclaw/memory/main.sqlite"
          },
          "cache": {
            "provider": "memory",
            "maxSessions": 128
          }
        }
      }
    }
  }
}

5. Use a SQLite path ​

Recommended path:

text
~/.openclaw/memory/main.sqlite

6. Restart OpenClaw ​

Config changes require a gateway restart.

The release bundle and npm package both include the recommended operator skill under skills/bamdra-memory-operator/, but current OpenClaw builds still require explicit agent.skills binding if you want that behavior layer active.

Validation checklist ​

  • Topic branches stay recoverable after interruption
  • Saved facts can be recalled later
  • Earlier decisions remain available after restart
  • The current prompt feels focused rather than bloated
