安装部署 ​

支持的安装方式 ​

这套插件支持两种公开安装路径：

  1. npm / OpenClaw CLI 安装
  2. release 压缩包手动安装

如果你的 OpenClaw 环境支持 npm 插件安装，优先推荐这条路径；如果你更偏好手动安装或离线路径，就用 release 包。

这 3 个插件都可以独立运行。通过 npm 安装 bamdra-openclaw-memory 时，如果本地 OpenClaw 扩展目录里还没有 bamdra-user-bind，主插件会自动把它补齐进去。bamdra-memory-vector 仍然保持独立、可选增强。

环境要求 ​

  • OpenClaw
  • Node.js 22 或以上
  • pnpm 10 或以上
  • 一个本地可写的 SQLite 路径

npm 安装 ​

bash
openclaw plugins install @bamdra/bamdra-openclaw-memory

这条安装路径还会自动创建本地 memory 目录，并在需要时把 bamdra-user-bind 自动补齐到 ~/.openclaw/extensions/。

然后在 ~/.openclaw/openclaw.json 里把 memory 和 contextEngine 都绑定到 bamdra-openclaw-memory。

如果你需要语义召回增强，再单独安装：

bash
openclaw plugins install @bamdra/bamdra-memory-vector

可直接下载的资源 ​

  • Bamdra OpenClaw Memory 正式发布包 v0.3.1
  • SQLite 本地配置样例
  • 完整套件配置样例
  • GitHub 首页
  • npm 包
  • bamdra-user-bind 仓库
  • bamdra-memory-vector 仓库

部署步骤 ​

1. 解压正式发布包 ​

压缩包中包含：

  • bamdra-openclaw-memory/
  • bamdra-user-bind/
  • bamdra-memory-vector/
  • examples/configs/
  • INSTALL.md

2. 准备 OpenClaw 目录 ​

主插件会在首次 bootstrap 时自动创建缺失的 memory 目录。手动安装时，你只需要确保 ~/.openclaw/extensions/ 存在即可。

3. 复制插件目录 ​

把下面这个目录复制到 ~/.openclaw/extensions/:

  • bamdra-openclaw-memory

可选增强：

  • bamdra-memory-vector

如果你希望 bamdra-user-bind 独立运行，也可以单独拷贝它；但它已经不再是安装主插件时必须手工执行的第二步。

4. 谨慎合并配置 ​

如果你当前的 OpenClaw 已经启用了其他插件，不要直接覆盖整个 plugins 对象。

为了兼容当前运行时，建议你的合并结果至少满足下面这些点：

  • plugins.enabled = true
  • plugins.allow 中包含 bamdra-openclaw-memory
  • plugins.load.paths 中包含统一插件目录
  • plugins.slots.contextEngine 和 plugins.slots.memory 都指向 bamdra-openclaw-memory
  • plugins.deny 中加入 memory-core
  • plugins.entries 中加入 bamdra-openclaw-memory 的配置项

推荐形态如下：

json
{
  "plugins": {
    "enabled": true,
    "deny": [
      "memory-core"
    ],
    "allow": [
      "bamdra-openclaw-memory"
    ],
    "load": {
      "paths": [
        "~/.openclaw/extensions/bamdra-openclaw-memory"
      ]
    },
    "slots": {
      "contextEngine": "bamdra-openclaw-memory",
      "memory": "bamdra-openclaw-memory"
    },
    "entries": {
      "bamdra-openclaw-memory": {
        "enabled": true,
        "config": {
          "enabled": true,
          "store": {
            "provider": "sqlite",
            "path": "~/.openclaw/memory/main.sqlite"
          },
          "cache": {
            "provider": "memory",
            "maxSessions": 128
          }
        }
      }
    }
  }
}

5. 设置 SQLite 路径 ​

推荐路径：

text
~/.openclaw/memory/main.sqlite

6. 重启 OpenClaw ​

配置更新后，需要重启 OpenClaw 网关。

release 压缩包和 npm 包都会附带推荐的 operator skill，路径是 skills/bamdra-memory-operator/，但当前 OpenClaw 版本如果要启用这层行为策略，仍然需要你在 agent.skills 中显式绑定。

验证是否成功 ​

如果安装正确，你应该能感受到：

  • 中断后仍能回到正确话题
  • 保存过的事实可以再次被引用
  • 重启后重要上下文不会彻底丢失
  • 当前 prompt 仍然保持紧凑
