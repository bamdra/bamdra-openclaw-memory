产品套件

三个插件，共同组成一套 continuity-first 记忆系统

Bamdra 现在以 3 个公开插件组成套件发布。它们都可以独立运行，而主记忆插件在安装时也可以自动补齐自己的身份依赖。

bamdra-openclaw-memory主记忆运行时，负责话题连续性、上下文装配、事实存储和长会话恢复。
bamdra-user-bind身份与画像插件，负责把原始 channel 身份转换成稳定用户边界。它可以独立运行，也可以在安装主记忆插件时被自动补齐。
bamdra-memory-vector可选的语义召回增强层，在不引入重型外部基础设施的前提下，为记忆体系补充 Markdown + 向量检索能力。
为什么要拆分分仓库、分 npm 包、分发布职责之后，安装逻辑更清晰，安全边界更可靠，后续迭代也更容易管理。

主入口插件

bamdra-openclaw-memory 是这套产品的主入口

bamdra-openclaw-memory 面向 OpenClaw 提供长期记忆能力。它能够分离话题分支、保存稳定事实、刷新摘要，并把当前上下文控制在更适合生产环境的体量里，让智能体在长会话中真正具备连续性。

bamdra-openclaw-memory 本身可以独立运行。通过 npm 安装时，如果本地 OpenClaw 扩展目录里还没有 bamdra-user-bind，主插件会自动把这个依赖补齐进去。

bamdra-memory-vector 则有意保持为独立、可选增强插件，只有你真的需要语义召回时才额外加上即可。

不是实验，而是工作中的协作能力
