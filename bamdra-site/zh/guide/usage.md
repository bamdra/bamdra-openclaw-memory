# 使用方式

## 正确的使用理解

`bamdra-openclaw-memory` 的目标，不是让用户频繁感知“工具被调用”，而是让 OpenClaw 在日常协作里自然具备连续性。在多数场景下，最好的体验是无感的：话题自动接回，稳定事实被持续利用，用户不需要重复背景。

## 两种使用模式

### 1. 无感连续性恢复

这是默认体验。用户只需要像往常一样在 OpenClaw 中工作，而记忆层会负责：

- 保留并恢复较早的话题分支
- 围绕当前任务装配更紧凑的上下文
- 在合适的时候复用之前保存过的稳定事实

### 2. 显式运维控制

当流程变得更复杂，或者 operator 需要更强控制时，可以直接调用这些记忆工具：

- `memory_list_topics`
- `memory_switch_topic`
- `memory_save_fact`
- `memory_compact_topic`
- `memory_search`

## 典型使用过程

### 正常协作中

用户会在规划、执行、跟进之间来回切换。记忆层的职责，是在不打断对话体验的前提下，保住真正重要的分支和事实。

### 当一个事实值得长期保存

对于偏好、环境路径、项目约束、决策结果等长期有效的信息，使用 `memory_save_fact` 保存是最合理的方式。

### 当需要更强控制时

如果团队需要有意识地回到某个分支，或者把某条线索单独管理，可以通过 `memory_list_topics` 和 `memory_switch_topic` 显式完成。

### 当某条分支已经很长

用 `memory_compact_topic` 刷新摘要，让后续恢复更准确、更清晰。

## 日常操作建议

- 普通连续性恢复优先交给隐式机制
- 只保存真正稳定、可复用的事实
- 在追问用户前，优先搜索记忆
- 让记忆帮助工作，而不是干扰对话体验
