# 最佳实践

## 用三层模型来理解它

最清晰的心智模型是：

- `bamdra-user-bind` 负责“这个人是谁”
- `bamdra-openclaw-memory` 负责“当前在做哪条工作分支”
- `bamdra-memory-vector` 负责“本地哪份知识最相关”

## 让知识库保持人类可维护

Markdown 应该是主要编辑面。

推荐这样理解：

- `knowledge/` 放稳定、可复用知识
- `docs/` 放正式文档和 changelog
- `notes/` 放会议纪要和过程性笔记
- `ideas/` 放未成熟但值得检索的想法

## 私有和共享一定分开

最好明确分成两套根目录：

- 私有知识
- 共享知识

这样检索才既有用又不模糊隐私边界。

## 本地召回应优先于 web search

对于本地 docs、changelog、SOP、笔记和 Markdown 知识，最佳顺序应该是：

1. 先本地召回
2. 本地结果够用就直接回答
3. 只有不够时再上网

这样既减少响应时间，也减少 token 浪费。

## 把画像镜像当作“活的用户说明书”

用户画像 Markdown 最适合保存：

- 应该怎么称呼
- 时区
- 语气偏好
- 角色与协作方式
- 重要的长期个人工作习惯

这才是稳定 per-user 个性化最合适的位置。

## 把整套系统当作会一起成长的东西

真正让人惊喜的地方，出现在长期使用之后：

- 画像越来越完整
- 记忆越来越可复用
- 知识库越来越有组织

这也是智能体随着用户一起进化的基础。
